# Git Toolkit - 远程脚本安装规格说明

## 概述

为 `git-toolkit` 提供一种通过远程脚本一键安装并启动的使用方式。用户可以通过 `curl` 或 `wget` 直接从 GitHub 下载并执行安装脚本，无需手动克隆仓库，实现即拿即用的体验。

---

## 用户故事

### 故事 1：新用户快速上手
作为一名首次听说 git-toolkit 的开发者，当我看到项目介绍中的一行安装命令时，我希望能够复制粘贴这行命令就能完成安装并立即开始使用，这样我就能快速体验工具的功能而不需要花费时间学习安装步骤。

### 故事 2：在新机器上快速部署
作为一名需要在新机器上配置开发环境的开发者，当我设置好一台新电脑时，我希望通过一条命令就能安装并启动 git-toolkit，这样我就能快速完成 Git 环境的配置，而不需要手动克隆仓库和配置 PATH。

### 故事 3：已安装用户更新版本
作为一名已经安装过 git-toolkit 的开发者，当我想更新到最新版本时，我希望再次运行远程脚本时能够检测到已安装并询问是否更新，这样我就能方便地保持工具是最新版本。

### 故事 4：网络不稳定时的可靠性
作为一名在网络不稳定环境下工作的开发者，当我运行安装脚本遇到网络问题时，我希望脚本能够自动重试几次，这样即使网络偶尔抖动也能成功完成安装。

## 功能性需求

### 1. 远程脚本入口
- **FR1.1** 提供一个可通过 `curl -fsSL <url> | bash` 执行的安装脚本
- **FR1.2** 同时支持 `wget -qO- <url> | bash` 方式
- **FR1.3** 脚本托管在项目仓库的 `install.sh` 文件中，通过 GitHub raw URL 访问

### 2. 系统检测与依赖检查
- **FR2.1** 检测操作系统类型（macOS 或 Linux）
- **FR2.2** 简单检查必要依赖：git、bash
- **FR2.3** 检查 curl 或 wget 是否至少有一个可用
- **FR2.4** 依赖缺失时给出清晰的提示信息

### 3. 安装流程
- **FR3.1** 检测是否已安装 git-toolkit（检查 ~/.git-toolkit 目录是否存在）
- **FR3.2** 如果已安装，提示用户并询问是否覆盖/更新
- **FR3.3** 从 GitHub 仓库主分支（main/master）下载最新代码
- **FR3.4** 将代码安装到 ~/.git-toolkit 目录
- **FR3.5** 下载失败时自动重试最多 3 次
- **FR3.6** 每次重试间隔 2 秒
- **FR3.7** 安装过程显示进度信息

### 4. 环境变量配置
- **FR4.1** 安装完成后询问用户是否需要自动配置环境变量
- **FR4.2** 检测用户使用的 shell（zsh 或 bash）
- **FR4.3** 如果用户同意，将 git-toolkit 的 bin 目录添加到 PATH
- **FR4.4** 在 shell 配置文件中添加标识块，便于后续管理
- **FR4.5** 配置完成后提示用户重新加载 shell 配置或重启终端

### 5. 启动功能
- **FR5.1** 安装成功后自动启动 git-toolkit 的交互式主菜单
- **FR5.2** 如果用户选择不配置环境变量，使用绝对路径启动
- **FR5.3** 如果用户选择配置环境变量，使用命令名启动

### 6. 错误处理
- **FR6.1** 所有操作失败时给出清晰的错误提示
- **FR6.2** 下载失败超过重试次数后，提示用户手动克隆仓库的方法
- **FR6.3** 安装过程中出现错误时，清理已创建的临时文件
- **FR6.4** 提供友好的退出码

## 非功能性需求

### 1. 可维护性
- **NFR1.1** 使用 Shell 脚本实现，遵循项目 constitution.md 中的脚本编写规范
- **NFR1.2** 代码结构清晰，逻辑分离
- **NFR1.3** 关键步骤有注释说明

### 2. 健壮性
- **NFR2.1** 启用 `set -euo pipefail` 严格模式
- **NFR2.2** 验证所有用户输入
- **NFR2.3** 使用 trap 清理临时资源
- **NFR2.4** 处理边界情况：目录已存在、权限不足、网络中断等

### 3. 可移植性
- **NFR3.1** 使用 POSIX 兼容的 Shell 语法
- **NFR3.2** 不硬编码绝对路径（除了 ~/.git-toolkit）
- **NFR3.3** 支持 macOS 和 Linux 系统
- **NFR3.4** 同时支持 zsh 和 bash

### 4. 易用性
- **NFR4.1** 整个安装过程尽可能自动化，减少用户输入
- **NFR4.2** 所有提示信息清晰友好
- **NFR4.3** 进度反馈明确，让用户知道当前在做什么
- **NFR4.4** 新手友好，包含必要的说明

### 5. 性能
- **NFR5.1** 安装过程快速，不包含不必要的操作
- **NFR5.2** 下载使用压缩或浅层克隆（如果可能）

```bash

# 使用 wget
wget -qO- https://raw.githubusercontent.com/username/git-toolkit/main/install.sh | bash
```
