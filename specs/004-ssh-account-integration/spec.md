# Git Toolkit - SSH 密钥与 Git 账号整合规格说明

## 概述

将 SSH 密钥管理与 Git 账号管理进行深度整合，实现概念统一，降低配置成本。用户在添加 Git 账号时可以一步完成 Git 信息和 SSH 密钥的配置，支持按域名关联不同密钥，同时保持向后兼容。

---

## 用户故事

### 故事 1：一键添加完整账号
作为一名开发者，当我需要添加新的 Git 账号时，我希望在一个流程中同时完成 Git 用户名、邮箱、SSH 密钥和域名的配置，这样我就不用分别操作多个功能，减少出错的可能。

### 故事 2：灵活选择 SSH 密钥
作为一名开发者，当我添加新账号时，我希望可以选择生成新的 SSH 密钥，或者选择已有的密钥，这样我可以根据实际情况灵活处理。

### 故事 3：按域名使用不同密钥
作为一名在多个平台有账号的开发者，我希望一个 Git 账号可以为不同的域名配置不同的 SSH 密钥，这样我可以用不同的密钥访问不同的 Git 平台，提高安全性。

### 故事 4：导入已有 SSH 密钥
作为一名已经有 SSH 密钥的开发者，我希望可以方便地将已有的 SSH 密钥关联到 Git 账号，这样我就不用重新生成密钥，保持现有配置的连续性。

### 故事 5：手动编辑配置兼容
作为一名喜欢手动编辑配置文件的开发者，我希望手动编辑配置文件后工具仍然能正常工作，这样我可以根据需要灵活调整配置。

## 功能性需求

### 1. 账号添加流程整合
- **FR1.1** 提供统一的账号添加交互流程，同时收集 Git 信息和 SSH 密钥信息
- **FR1.2** 在添加账号时提供 SSH 密钥选项：
  - 生成新的 SSH 密钥
  - 选择已有的 SSH 密钥
  - 暂不配置 SSH 密钥（后续可添加）
- **FR1.3** 生成新密钥时支持自定义密钥名称和注释
- **FR1.4** 选择已有密钥时列出 ~/.ssh 目录下的所有可用密钥供选择

### 2. 按域名关联不同 SSH 密钥
- **FR2.1** 账号配置支持为每个域名指定对应的 SSH 密钥
- **FR2.2** 如果未为特定域名指定密钥，使用账号的默认密钥
- **FR2.3** SSH config 生成时根据域名-密钥映射关系生成正确的配置
- **FR2.4** 编辑账号时支持修改域名与密钥的关联关系

### 3. SSH 密钥管理增强
- **FR3.1** 提供查看账号关联的所有 SSH 密钥的功能
- **FR3.2** 支持为已有账号添加/更换 SSH 密钥
- **FR3.3** 支持解除账号与 SSH 密钥的关联（不删除密钥文件）
- **FR3.4** 列出 SSH 密钥时显示被哪些账号/域名使用

### 4. 配置文件增强
- **FR4.1** 保持现有配置文件结构的向后兼容性
- **FR4.2** 支持新的配置格式：为每个域名指定密钥路径
- **FR4.3** 自动识别和兼容新旧两种配置格式
- **FR4.4** 手动编辑配置文件后工具仍能正常工作

### 5. 账号编辑功能增强
- **FR5.1** 编辑账号时可以修改 Git 信息（用户名、邮箱）
- **FR5.2** 编辑账号时可以修改 SSH 密钥配置
- **FR5.3** 编辑账号时可以修改域名列表及对应的密钥
- **FR5.4** 编辑账号时提供预览，显示将要做的变更

### 6. 通用功能
- **FR6.1** 所有操作提供清晰的成功/失败提示
- **FR6.2** 在执行破坏性操作前备份相关配置文件
- **FR6.3** 遵循项目现有代码风格和架构

## 输出格式示例

### 1. 新格式账号配置文件示例 (~/.git-toolkit/accounts/personal.conf)
```bash
ACCOUNT_NAME="personal"
GIT_USER_NAME="张三"
GIT_USER_EMAIL="zhangsan@example.com"
SSH_KEY_PATH="$HOME/.ssh/id_ed25519_personal"
DOMAINS=("github.com" "gitee.com")
DOMAIN_SSH_KEYS=(
  "github.com:$HOME/.ssh/id_ed25519_github"
  "gitee.com:$HOME/.ssh/id_ed25519_gitee"
)
```

### 2. 旧格式账号配置文件示例（向后兼容）
```bash

GIT_USER_NAME="张三"
GIT_USER_EMAIL="zhangsan@example.com"

```

### 3. SSH config 生成示例
```
# === git-toolkit managed start ===
Host github.com
  IdentityFile /Users/zhangsan/.ssh/id_ed25519_github
  IdentitiesOnly yes
  AddKeysToAgent yes
  UseKeychain yes

Host gitee.com
  IdentityFile /Users/zhangsan/.ssh/id_ed25519_gitee
  IdentitiesOnly yes
  AddKeysToAgent yes
  UseKeychain yes
# === git-toolkit managed end ===
```
