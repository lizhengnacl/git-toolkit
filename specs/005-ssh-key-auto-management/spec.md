# Git Toolkit - SSH 密钥自动化深度融合规格说明

## 概述

深度融合 Git 账号管理与 SSH 密钥管理，对用户只暴露 Git 账号管理界面，SSH 密钥完全由系统自动化管理。通过简化用户认知模型，降低使用和理解成本，同时为高级用户保留专家模式。

---

## 用户故事

### 故事 1：零认知添加 Git 账号
作为一名 Git 新手，当我需要添加 Git 账号时，我希望只输入用户名、邮箱和账号名称，系统自动处理所有 SSH 密钥相关的事情，这样我就不需要了解 SSH 密钥的概念也能正常使用 Git。

### 故事 2：灵活导入已有密钥
作为一名已经有 SSH 密钥的开发者，当我添加新账号时，我希望可以选择使用已有的 SSH 密钥，这样我就不用重新生成和配置密钥，保持现有工作流的连续性。

### 故事 3：删除账号时的安全提示
作为一名谨慎的用户，当我删除 Git 账号时，我希望系统询问我是否同时删除对应的 SSH 密钥，这样我可以避免误删重要的密钥文件。

### 故事 4：高级用户的专家模式
作为一名高级用户，当我需要手动管理 SSH 密钥时，我希望通过环境变量启用专家模式，这样我可以访问完整的 SSH 密钥管理功能，满足特殊需求。

### 故事 5：无缝升级体验
作为一名现有用户，当我升级到新版本时，我希望系统自动迁移我的旧配置，这样我不需要手动修改任何东西就能继续使用。

## 功能性需求

### 1. Git 账号添加流程简化
- **FR1.1** 添加账号时默认自动生成 SSH 密钥，无需用户手动操作
- **FR1.2** 提供选项让用户选择：自动生成密钥、选择已有密钥、暂不配置密钥
- **FR1.3** 自动生成的密钥使用 ed25519 类型，文件名为 `id_ed25519_<账号名>`
- **FR1.4** 密钥注释使用格式：`<账号名> <邮箱>`
- **FR1.5** 选择已有密钥时列出 `~/.ssh` 目录下的所有可用密钥
- **FR1.6** 如果选择的密钥已被其他账号使用，显示警告但允许继续

### 2. SSH 密钥管理隐藏化
- **FR2.1** 主菜单中移除 SSH 密钥管理选项
- **FR2.2** `git-toolkit ssh` 命令仍可调用，但不在帮助信息中显示
- **FR2.3** 添加账号成功后始终展示公钥内容供用户复制到 Git 平台
- **FR2.4** 列出账号时显示该账号使用的 SSH 密钥信息（只读）

### 3. 账号删除流程
- **FR3.1** 删除账号时询问用户是否同时删除对应的 SSH 密钥文件
- **FR3.2** 默认选择不删除 SSH 密钥文件
- **FR3.3** 如果选择删除，同时删除公钥和私钥文件
- **FR3.4** 删除前确认密钥确实属于该账号（防止误删共享密钥）

### 4. 专家模式
- **FR4.1** 通过环境变量 `GIT_TOOLKIT_EXPERT_MODE=true` 启用专家模式
- **FR4.2** 专家模式下，主菜单显示 SSH 密钥管理选项
- **FR4.3** 专家模式下，`git-toolkit ssh` 命令在帮助信息中正常显示
- **FR4.4** 专家模式下，所有 SSH 密钥管理功能完全可用

### 5. 配置迁移与兼容性
- **FR5.1** 首次运行新版本时自动检测旧格式配置
- **FR5.2** 自动将旧格式配置迁移到新格式
- **FR5.3** 迁移前自动备份旧配置
- **FR5.4** 保持向后兼容，仍支持读取旧格式配置

### 6. 通用功能
- **FR6.1** 所有操作提供清晰的成功/失败提示
- **FR6.2** 在执行破坏性操作前备份相关配置文件
- **FR6.3** 遵循项目现有代码风格和架构
